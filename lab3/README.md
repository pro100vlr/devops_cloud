CI/CD: плохой и хороший пример


| № | Плохая практика | Почему плохо | Как исправлено в `good-ci.yml` |
|---|----------------|--------------|-------------------------------|
| 1 | Использование `ubuntu-latest` | При обновлениях образа могут возникнуть неожиданные ошибки и несовместимости. | Указана фиксированная версия ОС `ubuntu-22.04`, что делает workflow стабильным и воспроизводимым. |
| 2 | Устаревшая версия `actions/checkout@v1` | Могут отсутствовать новые функции и исправления багов. | Используется последняя стабильная версия `actions/checkout@v4`. |
| 3 | Установка Node.js через `apt-get` | GitHub Actions уже содержит Node.js, установка через apt-get может привести к устаревшей версии. | Используется `actions/setup-node@v3` с конкретной версией Node.js (`20`). |
| 4 | Отсутствие таймаутов (`sleep 5`) | Бессмысленная задержка может замедлять выполнение, неэффективно. | Добавлен пример с `timeout 10 bash -c 'sleep 5'` — демонстрация таймаута для защиты от зависаний. |

---

- Плохой файл `bad-ci.yml` выполняется, но содержит потенциальные риски нестабильности и устаревшие методы.
- Хороший файл `good-ci.yml` исправляет все указанные плохие практики:
  - Workflow стал стабильным и воспроизводимым.
  - Используются актуальные версии инструментов.
  - Добавлен контроль времени выполнения команд.
