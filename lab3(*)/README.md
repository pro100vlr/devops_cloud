## Шаг 1.Развёртывание AppSec Auditor и Portal

- На первом этапе была выполнена установка и базовая проверка работоспособности AppSec Auditor.

- Репозиторий Auditor был склонирован из удалённого Git-репозитория.

- После клонирования сервис был запущен с использованием `docker-compose -d` 

- Через веб-интерфейс Auditor, доступный по адресу 127.0.0.1:8080, был сгенерирован Access Token.

- Полученный токен был добавлен в файл .env в каталоге Auditor в виде переменной окружения ACCESS_TOKEN.

- Для применения изменений контейнеры Auditor были перезапущены.

- После выполнения данных шагов сервис Auditor корректно запустился и стал доступен для взаимодействия.
